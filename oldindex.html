// playable learning loop

A cloud engineering game that still teaches real fundamentals

campaign.mode
// missions, rewards, progression
mode Campaign = separate_state()
Work through levels from basic landing zones to VNet peering, IIS, SQL, Active Directory and operations boss fights. Complete objectives to earn credits.
sandbox.mode
// freeplay cloud lab
mode Sandbox = experiment()
Build whatever you want using the Portal-style GUI or Terraform CLI. Sandbox state stays separate from Campaign so experiments never ruin missions.
network.rules
// realistic connectivity checks
test(vnet, subnet, nsg, route)
VMs only talk when the design allows it. Peer VNets, attach NSGs, route through firewalls, debug DNS, and use tflab test to learn why traffic fails.
vm.desktop
// connectable servers
connect-gui vm-web-01
Open a simulated Windows Server desktop, launch CMD from Start, install IIS, create SQL databases, promote domain controllers and join machines to AD.
portal.create
// Azure-style creation forms
create(resource) → main.tf
Create resources from category-aware GUI menus with expected dependencies like VNet/subnet/NIC/NSG choices, while TFLab writes the Terraform for you.
wizard.loot
// fun layer
open(crate) + zap(monsters)
Earn credits, open wizard crates, unlock cosmetics, and click Terraform-themed monsters like drift gremlins and syntax bats when they invade the lab.
